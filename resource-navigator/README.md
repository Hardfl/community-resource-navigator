# Community Resource Navigator

A Claude-powered tool that helps nonprofit caseworkers search, summarize, and recommend community resources for clients in need. Built for organizations working in housing, food security, environmental services, humanitarian aid, and public health.

*Resource data sourced from 211.org, findhelp.org, and direct organization websites for Middlesex County, NJ. In production, this would route through a backend to protect the API key...

---

## What it does

- **Smart search** — caseworkers describe a client's situation in plain language; Claude finds the best matching resources and explains why
- **Intake notes** — clicking any resource card triggers a second Claude call with practical notes for the caseworker (eligibility flags, call-ahead tips)
- **Impact dashboard** — tracks search volume, match rates, and unmet needs over time
- **Gap analysis** — automatically surfaces categories where searches consistently return zero results, helping leadership prioritize new partnerships

## Setup

### 1. Clone the repo

```bash
git clone https://github.com/YOUR_USERNAME/resource-navigator.git
cd resource-navigator
```

### 2. Get a Claude API key

1. Go to [console.anthropic.com](https://console.anthropic.com)
2. Sign in and click **API Keys** in the left sidebar
3. Click **Create Key**, give it a name like `resource-navigator`
4. Copy the key — you will only see it once

### 3. Add your API key to the app

Open `index.html` in a text editor. Find this section near the bottom of the file (inside the `<script>` tag):

```javascript
const res = await fetch('https://api.anthropic.com/v1/messages', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'x-api-key': 'YOUR_API_KEY_HERE',   // ← paste your key here
    'anthropic-version': '2023-06-01',
    'anthropic-dangerous-direct-browser-access': 'true'
  },
```

> ⚠️ **Important:** This setup is for local use only. If you ever host this on a public server, you must move the API key to a backend. Never commit your key to GitHub — it's in `.gitignore` by default but double-check before pushing.

### 4. Open the app

Double-click `index.html` or open it in Chrome/Firefox. No server needed.

## Running the data pipeline

The Python pipeline cleans raw CSV data and loads it into a local SQLite database. It also generates a gap analysis report.

### Install dependencies

```bash
pip install -r requirements.txt
```

### Run the pipeline

```bash
python scripts/clean_and_load.py
```

This will:
- Read `data/resources_raw.csv`
- Print a data quality report to the terminal
- Create `resources.db` (SQLite database)
- Export `data/resources_clean.csv`
- Print and save a gap analysis to `data/gap_report.txt`

### Set up the database directly (optional)

If you want to explore the database with SQL:

```bash
sqlite3 resources.db < sql/schema.sql
sqlite3 resources.db
```

Then try some of the example queries at the bottom of `sql/schema.sql`.

## Adding or updating organizations

### Option A — Edit the CSV (recommended for non-technical staff)

1. Open `data/resources_raw.csv` in Excel or Google Sheets
2. Add a new row for each organization or service
3. Required columns: `org_name`, `category`, `service_description`, `eligibility`, `phone`, `hours`
4. Valid categories: `housing`, `food`, `health`, `environment`, `humanitarian`
5. Save the file
6. Run the pipeline: `python scripts/clean_and_load.py`
7. Reload `index.html` in your browser

### Option B — Edit the database directly (for technical staff)

```sql
-- Add a new organization
INSERT INTO organizations (name, address, phone, languages, last_verified)
VALUES ('New Org Name', '123 Main St', '555-0100', 'English,Spanish', '2025-01-15');

-- Add a service for that org
INSERT INTO services (org_id, category, service_name, description, hours, capacity)
VALUES (last_insert_rowid(), 'housing', 'Emergency Shelter', 'Overnight shelter for adults', '24/7', '20 beds');
```

## Useful SQL queries

These are included in `sql/schema.sql` as comments. Copy and run them in `sqlite3 resources.db`.

**All active services by category:**
```sql
SELECT o.name, s.category, s.service_name, s.hours, s.capacity
FROM services s
JOIN organizations o ON s.org_id = o.org_id
WHERE o.is_active = 1 AND s.is_active = 1
ORDER BY s.category, o.name;
```

**Top unmet needs (last 30 days):**
```sql
SELECT query, COUNT(*) AS searches
FROM search_logs
WHERE matched = 0 AND searched_at >= DATE('now', '-30 days')
GROUP BY LOWER(query)
ORDER BY searches DESC
LIMIT 20;
```

**Coverage rate by category:**
```sql
SELECT category_filter,
       COUNT(*) AS total,
       ROUND(100.0 * SUM(matched) / COUNT(*), 1) AS match_rate_pct
FROM search_logs
WHERE category_filter IS NOT NULL
GROUP BY category_filter
ORDER BY match_rate_pct ASC;
```

**Organizations with stale data (not verified in 90+ days):**
```sql
SELECT name, last_verified,
       CAST(julianday('now') - julianday(last_verified) AS INTEGER) AS days_since_verified
FROM organizations
WHERE is_active = 1
  AND (last_verified IS NULL OR julianday('now') - julianday(last_verified) > 90)
ORDER BY days_since_verified DESC;
```

## Staff runbook

### Who owns this tool?

Assign one staff member as the **resource data owner**. Their job is to:
- Review the gap report monthly and share findings with leadership
- Keep `resources_raw.csv` up to date when organizations change hours, capacity, or eligibility
- Run `python scripts/clean_and_load.py` after any CSV update
- Verify 2–3 organizations per week by phone (mark `last_verified` in the CSV)

### Monthly maintenance checklist

- [ ] Run `python scripts/clean_and_load.py` and review the quality report output
- [ ] Open `data/gap_report.txt` and share top unmet needs with your director
- [ ] Call any organization flagged as "stale" (not verified in 90+ days) to confirm their info
- [ ] Update `last_verified` column in `resources_raw.csv` after confirming

### Common issues

**"No results found" for common needs**
This usually means either (a) no organization covers that need — this is real gap data, save it in the gap report — or (b) the search terms don't match how the resource is described in the CSV. Try updating the `service_description` field to include synonyms.

**The app isn't calling Claude**
Check that your API key is correctly pasted in `index.html`. Open your browser's developer tools (F12) → Console tab to see any error messages.

**Pipeline fails with "file not found"**
Make sure you're running the script from the project root directory, not from inside `scripts/`. Use: `python scripts/clean_and_load.py` not `python clean_and_load.py`.
